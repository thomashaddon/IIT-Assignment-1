Project Idea

Overview

For this project, the idea is to create an analytical tool for the game Counter Strike: Global Offensive(CSGO) which shows the heatmap of player activity in the map de_mirage. To explain what CSGO is, CSGO is a 30 round(first to 16) 5v5 competitive objective based tactical shooter. In the game there are two opposing teams composed of the counter-terrorists (CT’s) and terrorists (T’s). The objective of the terrorists is to plant and detonate a bomb at one of two specified bomb sites. The objective of the counter-terrorists is to prevent that from happening. The objective of this project is to show it’s users the general player tendencies of where players from both team’s interact with one another, where team fights happen and what utility is most frequently utilised throughout a round. From this data the player can make a more informed decision when either attacking a site or which site to focus their defenses.

Motivation

The motivation for this project is to help more new players starting the game to learn the game more efficiently using the data to make better decisions in game. On average in the last 30 days CS GO has over 749,847 players(Steamcharts,2012) and is continually growing steadily every month. This indicates that there is a market for new players joining the community of players every month and may perhaps would like to receive some help when starting out the game. This tool may be used as a stepping stone to help these players get started and learn the basics of where players tend to go and help them learn a new map faster.

Description

The features of this tool will include as previously stated a heatmap showing where both sides of CT’s and T’ engage one another, showing the tendencies of where CT’s prefer to defend on the specified areas on a site and where the T’s prefer to attack from. This tool will also take data of which side uses more specific utility such as grenades, smokes, flashes and in which specific time in the game. CS GO uses an in game economy system to manage weapons and utility for both sides. This means the more a team wins rounds the more money in the economy will be available to that specific side. However to counteract this a in-game feature known as a loss bonus is given to the team that loses the round and stacks up to 4 rounds. The tool should be able to show what are the preferred purchases during the pistol round(this is the first round of each half), a half buy(buying rifles and armour but no grenades), a full buy(buying armour, rifles, and grenades).

Using a heatmap can show where a side would fail most often giving the player an idea of where he or she would most likely fail if they were to defend a site from a specific position or T’s where they would most likely fail if they were to attack from a certain route. Another feature of this tool would be to inform the CT player if they should be buying a defuse kit or not (a defuse kit reduces the time of defusing the bomb from 5 seconds to 10 seconds). This is a crucial piece of utility for CT players as many games have been won or lost depending if the player bought a defuse kit or not. This would mean the tool would also show when during the game the bomb is most likely to be planted.

However the question is what level of skill should be used as data for this tool. In the game there are over 18 ranks, the lowest being involving the newest players and the highest rank including players that have put in countless hours into the game. The data should be taken from a wide range of players from every rank to give the most accurate data of player tendencies. Perhaps another feature would be to show the user specific rank heatmaps as more experienced players would play the game differently compared to newer players.

Tools and Technologies

The software to be used would be python and CSGO, the python script would intake the data from CS GO rounds and compile the heat maps onto a map of de_mirgae showing the user visually the data. The hardware that would be used is any computer that can be powerful enough to run python scripts and can compile thousands upon thousands of rounds of Counter Strike as this would be necessary if the data of over 18 ranks were to be taken from.

The skills required for this project would be proficiency in python, this can be developed over time during the duration of the course and is further developed in RMIT’s introduction to programming which teaches the course in python. The feasibility of this is rather probable as assignment 3 is due in 2 months and over the course of 2 months my understanding of python will be more defined.

Outcome

The outcome of this project if successful there will be a program that can show users a heatmap of the map de_mirage in CS GO and other useful data that further informs the player to make better decisions when playing the game. This will be a tool very useful for new players learning maps in this case being de_mirage. The impact of this would be rather insignificant as for the majority of players playing CS GO have already been playing for many years and this project would be aimed at a niche demographic.
